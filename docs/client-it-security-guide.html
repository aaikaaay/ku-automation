Option C: On-Premises Deployment

AIR-GAPPED

Full deployment within your data centre or private infrastructure. Complete control over all components with no external dependencies.

Your Requirements
Bare-metal or VM infrastructure; Docker/Podman or Kubernetes; internal PKI for certificates
Infrastructure Specs
Minimum: 8 vCPU, 32 GB RAM, 500 GB storage; recommended: 3-node HA cluster
Security
Fully air-gapped operation supported; your HSM/key management; full audit log control
Best For
Defence/government; highly regulated environments; zero external connectivity requirements

Option D: Hybrid Deployment

FLEXIBLE

Combine cloud-hosted AI processing with on-premises data storage. Sensitive data never leaves your network while leveraging cloud AI capabilities.

Your Requirements
On-prem gateway server; secure tunnel (VPN/Private Link) to KU cloud; outbound HTTPS
Infrastructure Specs
Gateway: 2 vCPU, 8 GB RAM, 50 GB storage; VPN appliance or SD-WAN connectivity
Security
Data anonymisation at gateway; customer encryption keys for data at rest; split processing
Best For
Organisations wanting cloud AI benefits while maintaining data sovereignty

🔒 Security Requirements

KU Automation is designed with enterprise security requirements in mind. The following security controls should be implemented based on your deployment option and organisational policies.

🔐

Authentication

SAML 2.0 / OIDC SSO integration supported. Connect to your identity provider (Okta, Azure AD, Google Workspace, Ping). MFA enforcement recommended.

🛡️

Network Security

All traffic encrypted with TLS 1.2+ (TLS 1.3 preferred). IP allowlisting available. VPC peering / Private Link for cloud deployments.

🗄️

Data Encryption

AES-256 encryption at rest. Customer-managed keys (BYOK) supported for Private/On-Prem. TLS in transit for all data flows.

📝

Audit Logging

Comprehensive audit logs for all user actions and system events. SIEM integration via syslog, webhook, or API. Configurable retention.

👥

Access Control

Role-Based Access Control (RBAC) with predefined roles (Admin, Operator, Viewer). Custom roles available. Principle of least privilege enforced.

🔍

Vulnerability Management

Regular security patches and updates. Container images scanned with Trivy/Clair. Responsible disclosure program in place.

4.1 Security Configuration Checklist

Work with your security team to complete the following before go-live:

  • TLS Certificate: Provision or verify TLS certificate for the application endpoint (valid chain, 2048-bit+ RSA or ECC)
  • SSO Integration: Configure SAML/OIDC integration with your identity provider; test user provisioning
  • Network Rules: Update firewall/security groups to allow required traffic (see Section 3.1); block all unnecessary ports
  • WAF Rules: Enable web application firewall with OWASP Core Rule Set; configure rate limiting
  • Encryption Keys: Generate or provide encryption keys for data at rest (if using BYOK)
  • Audit Log Destination: Configure SIEM integration endpoint for audit log forwarding
  • Backup Policy: Confirm backup schedule and retention (recommend: daily incremental, 30-day retention)
  • IP Allowlisting (Optional): Provide list of allowed source IPs/ranges for additional access restriction
  • Private Link (Optional): Configure VPC peering or Private Link for private connectivity
