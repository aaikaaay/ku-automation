## What a "Detailed Review" Actually Returns

A Detailed review is the full two-pass output. Quick reviews ($19) cover code compliance and headline issues; Detailed reviews ($39–$79 depending on discipline) add an internal critique pass, an executive summary, and an action register sized for an IFR meeting.

For a single P&ID, a typical Detailed review returns:

- **Executive summary** — recommendation, top three issues, and the reviewer's overall confidence in the deliverable as-issued
- **Findings register** — every finding tagged Critical / Major / Minor, with the exact code citation (e.g. *ASME B31.3 §304.5.3*, *API 520 Part I §5.3.2*, *ISA 5.1 §6.4*, *NACE MR0175 §A.2.1.4*), a description of the deviation, the consequence if uncorrected, and the recommended remediation
- **Action register** — every finding mapped to a Required-Before-IFA, Required-Before-IFC, or Pre-Construction status with a suggested responsible discipline
- **PDF report** — formatted for circulation to the EPC and DNO/owner project teams
- **Excel comment register** — multi-sheet workbook that drops straight into your existing review-comment workflow

For a pump datasheet, you get the same structure tuned to API 610 / API 674–676 / API 682 / NACE MR0175. For a pipe rack, AISC 360 / IBC / ASCE 7. For a compressor datasheet, API 617 / API 618 / API 619 depending on type. Every reviewer ships with its own discipline-specific system prompt and checklist — none of this is generic LLM output dressed up in a template.

## What "Two Free" Lets You Actually Validate

Two Detailed reviews is enough to run a credible internal pilot. Here are the four patterns we have seen new accounts use in the first week of the trial gate:

**Pattern 1 — Run the same deliverable through two disciplines.** Upload an isometric to the Piping Isometric reviewer ($29 Detailed normally) and the matching pipe support drawing to the Structural Pipe Support reviewer ($39). This is the cleanest way to see how the AI catches cross-discipline misalignment — a support tagged for a 6" Sch 40 line that the isometric has rerouted as 8".

**Pattern 2 — Pit the AI against a known reference.** Pick a deliverable you have already redlined. Run it through the matching reviewer. Compare. We have had Owners' Engineers find that the AI catches every comment they raised plus three they did not — typically code references they assumed someone else was checking. We have also had reviewers whose own comments looked stronger than the AI's. Both outcomes are useful. Both inform whether and how to use the marketplace going forward.

**Pattern 3 — Stress-test on a project you do not have time to review.** Most owner-engineering teams have a backlog of vendor drawings sitting at "received, not yet reviewed". Drop two of them in. See whether the output is fit to forward to the EPC with your own cover note, or whether it points to issues you would have flagged differently.

**Pattern 4 — Compare disciplines on the same project basis.** Run a P&ID and the matching process calculation through Process AI. Run a structural foundation drawing and the matching steel structure through Structural AI. See whether the reviewers stay consistent across deliverables of the same project — they should, because the prompts share the same code library and project safety basis assumptions.

Either pattern gives you a defensible answer to *should we use this*? — without paying for the privilege of finding out.

## What the Two-Review Limit Is Not

The trial gate is two **reviews**, not two **uploads**. A revision review (where you upload a corrected version of an already-reviewed deliverable and ask for a comparison against the previous findings register) counts as one review. A multi-sheet P&ID set submitted as a single document counts as one review regardless of sheet count. We do not throttle by file size, page count, or discipline mix within the trial.

The trial is also not time-limited. You can run review one today, sit on the result for three weeks, and run review two when a deliverable you actually want to test arrives. The two reviews are yours.

What the trial **does not** unlock is workflow automation. Recurring vendor-document control, cross-deliverable consistency sweeps across an entire IFR package, and custom code-library overrides are paid features sized to project scope — those start with a short scoping call after the trial has been used.
